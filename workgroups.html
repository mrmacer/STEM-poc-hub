STEM INTELLIGENCE LAYER

The workgroups generate more than meeting notes. They generate statewide implementation intelligence: recurring needs, active projects, shared resources, leadership patterns, action items, and evidence of impact. This layer turns workgroup activity into something searchable, useful, and strategic.

From meeting archive → living knowledge system.
WHAT THIS LAYER DOES

Detects Patterns

Surfaces repeated needs across workgroups — STEELS implementation, educator overwhelm, assessment support, AI/CS integration, and resource navigation.

Connects Work

Links workgroup projects to deliverables, ORBIT evidence, resource hubs, and statewide priorities so nothing stays siloed.

Preserves Memory

Captures what each workgroup has learned so the network does not lose momentum when roles, grants, or leadership change.

Turns Notes Into Action

Converts meeting minutes into active projects, next steps, resource needs, and impact stories the network can act on.

STATEWIDE SIGNALS EMERGING FROM WORKGROUPS

STEELS Is the Center of Gravity

Many groups orbit around STEELS implementation, assessment, standards translation, administrator support, classroom implementation, and teacher readiness. It is the connective thread across most deliverable areas.

Translation Is the Pain Point

The network has resources, but educators need clearer pathways: "What do I need right now?" "Where do I start?" "What does this look like in my context?" Navigation and personalization are the missing layer.

AI, CS, and CT Are Becoming Cross-Curricular

Computer Science work is expanding beyond coding into AI literacy, computational thinking, cybersecurity, drones, data, and support for non-CS teachers. The audience and entry points are widening.

Equity Must Be Embedded, Not Isolated

Equity, TaC, UDL, accessibility, special education, and multilingual learner supports need to be threaded through every workgroup — not held in a single silo that others can opt out of.

Ecosystems Are Becoming Strategic Infrastructure

Ecosystem work is shifting from "partnerships are nice" to regional engines for workforce engagement, industry panels, community mini-grants, judge recruitment, and statewide storytelling.

Workgroups Need Memory Systems

Multiple groups are building resources year over year, but the network needs better ways to preserve decisions, action items, links, and impact stories across leadership transitions and grant cycles.

FUTURE BUILD: WORKGROUP PROFILE PAGES

The hub can evolve in stages as workgroup data matures and the network decides how deep each profile should go.

Option A — Recommended Now

Stay Single-Page

Best for: simple maintenance and fast updates

Each workgroup has expandable intelligence cards inside the single HTML file. No build system, no backend, no deployment pipeline needed.

Option B — Next Stage

One HTML Per Workgroup

Best for: richer profiles and shareable links

Create /workgroups/science.html, /workgroups/cs.html, etc. Allows deeper content per group without bloating the main hub file.

Option C — Long-Term

Data-Driven JSON System

Best for: long-term scalability and dynamic rendering

Store workgroup data in a JavaScript object or JSON file and render cards dynamically. Enables search, filters, and profile updates without touching HTML structure.

Current recommendation: Use Option A now. Move to Option C when workgroup profiles need dedicated pages, richer data, or contributor editing access.
